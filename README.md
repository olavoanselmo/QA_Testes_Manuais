# QA_Testes_Manuais
Estudo sobre os testes manuais em qualidade de software.


1. O que são testes manuais?

O teste manual é o teste feito manualmente por um analista, desenvolvedor ou especialista em qualidade. Nessa situação, a pessoa responsável pelos testes irá executar cada passo necessário para que o teste seja realizado com sucesso, sempre atento para as condições que o teste precisa para ser realizado da forma correta.

O teste manual costuma ter baixo valor de investimento e também permite que a pessoa que os realiza, experimente condições semelhantes às do ambiente de produção, já que o QA pode definir os parâmetros do teste manualmente.

Tem como desvantagem serem mais lentos e como dependem totalmente da interação humana, sempre existe uma alta possibilidade de um problema passar despercebido por quem testa.

🧪 2.Diferenças basicas entre Teste Manual e Teste Automatizado

🔹 Teste Manual
Definição: O tester executa os testes manualmente, passo a passo, sem usar scripts ou ferramentas de automação.

Características:

Baseado em interação humana.
os testes manuais são ideais para situações de mudança frequente, bem como para testes exploratórios, usabilidade.
Exige mais tempo para realização de cada teste.
Pode ser mais propenso a erros humanos.
Exemplo:
Você acessa a página de login, digita o e-mail e a senha, clica em "Entrar" e observa o comportamento do sistema.

🔸 Teste Automatizado
Definição: Os testes são executados automaticamente por meio de scripts e ferramentas, sem precisar de interação humana após estarem configurados.

Características:

Baseado em código/script.
Ideal para testes repetitivos, de regressão ou com grande volume.
Mais rápido e escalável.
Requer um maior conhecimento técnico (linguagens, ferramentas como Selenium, Selenium, Cypress, Playwright etc.).
Exemplo:
Você escreve um script que acessa a página de login, insere e-mail e senha automaticamente, clica em "Entrar" e verifica se a página redirecionou corretamente.


# 🧪 Testes Manuais vs. Testes Automatizados: Guia de Memorização

Este guia visual resume as principais diferenças, vantagens e cenários de aplicação para testes manuais e automatizados em Qualidade de Software (QA).


## 💡 Gatilhos de Memorização (Cheat Sheet)

*   **Manual = Explorar.** Pense em um **humano** usando a intuição para caçar bugs em áreas novas e avaliar o visual.
*   **Automatizado = Repetir.** Pense em um **robô** executando tarefas repetitivas e pesadas (regressão) com velocidade máxima.
*   **A Regra do Custo:** O manual é barato para começar, mas fica caro no longo prazo (tempo/pessoas). O automatizado é caro para começar, mas se paga no longo prazo.
